# 项目愿景

> 统一 AI 工具的 Skill 生态，让开发者在不同 AI 工具间无缝切换。

---

## 背景问题

当前市面上的 AI 开发工具（如 OpenClaw、OpenCode、ClawHub 等）各自维护独立的 Skill 体系，导致：

- **Skill 无法复用** — 在 OpenClaw 编写的 Skill，到了 OpenCode 或 ClawHub 无法直接使用
- **重复开发** — 同一功能需要在多个平台重复实现，造成资源浪费
- **生态割裂** — 开发者被绑定在单一工具上，难以尝试更好的方案
- **学习成本高** — 每个工具都有不同的 Skill 规范和目录结构

---

## 我们的解决方案

AgentHub 致力于构建**跨平台的统一 Skill 规范**，实现：

### 1. 统一 Skill 格式
所有支持的 AI 工具使用相同的 Skill 规范（v2.0），包括：
- 统一的 YAML frontmatter 结构
- 统一的触发词（triggers）机制
- 统一的元数据（metadata）格式

### 2. 跨平台兼容
一套 Skill，多端通用：
- OpenClaw ✅
- OpenCode ✅
- ClawHub ✅
- Hermes Agent ✅
- 其他兼容工具（规划中）

### 3. 共享技能库
- 集中管理 Skill，避免重复建设
- 社区共建，形成良性生态
- 质量有标准，版本可追溯

---

## 核心价值

| 价值 | 说明 |
|------|------|
| **降本增效** | 一次编写，到处运行 |
| **生态开放** | 不被单一工具绑定 |
| **社区共建** | 集体智慧，持续迭代 |
| **标准引领** | 推动 AI 开发工具规范化 |

---

## 目标用户

- **AI 工具开发者** — 希望自己的工具有丰富的 Skill 生态
- **AI 应用开发者** — 希望Skill能在不同平台间复用
- **技术爱好者** — 希望探索和对比不同 AI 工具

---

## 未来展望

1. **更多平台支持** — 随着项目发展，支持更多 AI 开发工具
2. **Skill 市场** — 建立开放的 Skill 分享平台
3. **标准化推进** — 推动行业采用统一的 Skill 规范
