# AgentHub TODO

> 版本: v2.0 | 更新: 2026-04-28
> 完整规范: [00-TODO-SPEC.md](./00-TODO-SPEC.md)

## 概述

AgentHub TODO 是一个简单的任务追踪系统，用于记录和追踪 Agent 工作过程中的待办事项。

**规范版本**: v1.1 — 对齐 AgentHub 社区标准（新增作业分类）

## 目录结构

```
.agenthub/TODO/
├── README.md              # 本文件
├── todos.json            # JSON 数据
├── [归档]/               # 已完成的任务
│   └── [归档]001-xxx.md
├── [待办]001-xxx.md      # 待开始
├── [进行中]002-xxx.md    # 进行中
└── [作业]003-xxx.md      # 作业任务
```

## 状态显示

| 前缀 | 状态 | 位置 |
|------|------|------|
| `[待办]` | 待开始 | 根目录 |
| `[进行中]` | 进行中 | 根目录 |
| `[作业]` | 作业任务 | 根目录 |
| `[归档]` | 已完成 | `[归档]/` 文件夹 |

## 文件命名规则

```
[{状态前缀}]{序号}-{标题}.md
```

示例：
- `[待办]001-科目一学习.md`
- `[进行中]002-博客优化.md`
- `[作业]003-高数课后习题.md`
- `[归档]004-旧任务.md`

## 单个待办条目格式

```markdown

priority: high
```

## 状态流转

```
[待办] ──开始──▶ [进行中] ──完成──▶ [归档]
  │                    │
  └──取消──────────────┴──────▶ [归档]

[作业] ──开始──▶ [进行中] ──完成──▶ [归档]
  │                    │
  └──取消──────────────┴──────▶ [归档]
```

## AI 使用规范

### 何时创建 Todo

| 场景 | 示例 | 是否创建 |
|------|------|----------|
| 用户明确安排的任务 | "明天完成这个" | ✅ |
| AI 自己承诺的里程碑 | "我会在第三步完成后..." | ✅ |
| 需要多步骤完成的任务 | "这个需求需要5步" | ✅ |
| 一次性简单任务 | "改个变量名" | ❌ |

### 优先级判断

| 优先级 | 使用场景 |
|--------|----------|
| high | 用户明确要求、有截止日期、阻塞后续工作 |
| medium | 合理安排、重要但不紧急 |
| low | 可选功能、有时间再做 |

### 操作说明

| 操作 | 操作 |
|------|------|
| 创建待办 | 在根目录创建 `[待办]XXX.md` |
| 开始任务 | 重命名为 `[进行中]XXX.md` |
| 完成 | 移动到 `[归档]/` 目录，重命名为 `[归档]XXX.md` |
| 取消 | 移动到 `[归档]/` 目录 |
