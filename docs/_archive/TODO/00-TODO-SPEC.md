# TODO 规范文档 v1.1

> 版本: v1.1 | 创建: 2026-04-28 | 更新: 2026-04-28 | 适用: AgentHub/xavier

---

## 1. 概述

TODO 是 AgentHub 的任务追踪模块，用于 AI 记录和追踪待办事项。每个 TODO 是一个自包含的 Markdown 文件，状态通过文件名前缀和目录位置区分。

---

## 2. 目录结构

```
TODO/
├── README.md                  # 本文件
├── todos.json                 # JSON 数据索引（可选）
├── [归档]/                    # 已完成的任务
│   └── [归档]001-xxx.md
├── [待办]001-xxx.md           # 待开始
├── [进行中]002-xxx.md         # 进行中
└── [作业]003-xxx.md           # 作业任务
```

### 2.1 目录说明

| 目录/文件 | 说明 |
|-----------|------|
| `README.md` | 规范说明文档 |
| `todos.json` | JSON 格式的任务索引（可选） |
| `[归档]/` | 已完成/已取消的任务 |
| `[待办]*.md` | 待开始的任务（根目录） |
| `[进行中]*.md` | 进行中的任务（根目录） |
| `[作业]*.md` | 作业任务（根目录） |

---

## 3. 状态定义

| 状态前缀 | 含义 | 位置 |
|----------|------|------|
| `[待办]` | 待开始 | 根目录 |
| `[进行中]` | 进行中 | 根目录 |
| `[作业]` | 作业任务 | 根目录（独立分类） |
| `[归档]` | 已完成/已取消 | `[归档]/` 目录 |

---

## 4. 文件命名规则

### 4.1 命名格式

```
[{状态前缀}]{序号}-{标题}.md
```

### 4.2 规则说明

| 组成部分 | 规则 | 示例 |
|----------|------|------|
| 状态前缀 | 中文方括号 | `[待办]`、`[进行中]`、`[作业]`、`[归档]` |
| 序号 | 3位数字，零填充 | `001`、`002`、`003` |
| 分隔符 | `-` | `-` |
| 标题 | 中文/英文，无特殊字符 | `科目一学习`、`blog优化` |

### 4.3 示例

```
[待办]001-科目一学习.md
[进行中]002-博客优化.md
[作业]003-高数课后习题.md
[归档]004-旧任务完成.md
```

## 5. TODO 文件格式

### 5.1 YAML Frontmatter（必需）

```yaml
---
id: "001"
title: "科目一学习计划"
status: "pending"              # pending / in_progress / completed / cancelled
priority: "high"              # high / medium / low
created_by: "[dev-agent]"
created_at: "2026-04-20T10:00:00Z"
updated_at: "2026-04-28T15:30:00Z"
---
```

### 5.2 必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 序号，3位数字 |
| `title` | string | 任务标题 |
| `status` | enum | 状态 |
| `priority` | enum | 优先级 |
| `created_by` | string | 创建者 ID |
| `created_at` | string | ISO 8601 时间 |
| `updated_at` | string | ISO 8601 时间 |

### 5.3 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `parent` | string | 父任务 ID |
| `children` | string[] | 子任务 ID 列表 |
| `tags` | string[] | 标签数组 |
| `due` | string | 截止日期 |
| `dependencies` | string[] | 依赖的任务 ID |

### 5.4 状态枚举

| status | 说明 |
|---------|------|
| `pending` | 待开始 |
| `in_progress` | 进行中 |
| `homework` | 作业任务 |
| `completed` | 已完成 |
| `cancelled` | 已取消 |

### 5.5 分类标签（tags）

作业任务应添加 `homework` 标签：

```yaml
tags: ["homework", "课程名"]
```

### 5.5 优先级枚举

| priority | 说明 |
|----------|------|
| `high` | 高优先级，用户明确要求/有截止日期 |
| `medium` | 中优先级，合理安排 |
| `low` | 低优先级，可选功能 |

## 7. 状态流转

```
[待办] ──开始──▶ [进行中] ──完成──▶ [归档]
  │                    │
  └──取消──────────────┴──────▶ [归档]

[作业] ──开始──▶ [进行中] ──完成──▶ [归档]
  │                    │
  └──取消──────────────┴──────▶ [归档]
```

### 7.1 操作规则

| 操作 | 文件操作 |
|------|----------|
| 创建待办 | 在根目录创建 `[待办]XXX.md` |
| 创建作业 | 在根目录创建 `[作业]XXX.md` |
| 开始任务 | 重命名 `[待办]→[进行中]` 或 `[作业]→[进行中]` |
| 完成任务 | 移动到 `[归档]/`，重命名 `[进行中]→[归档]` |
| 取消任务 | 移动到 `[归档]/`，状态改为 `cancelled` |

## 9. AI 使用规范

### 9.1 何时创建 TODO

| 场景 | 示例 | 是否创建 |
|------|------|----------|
| 用户明确安排的任务 | "明天完成这个" | ✅ |
| AI 自己承诺的里程碑 | "我会在第三步完成后..." | ✅ |
| 需要多步骤完成的任务 | "这个需求需要5步" | ✅ |
| 一次性简单任务 | "改个变量名" | ❌ |
| 用户明确说不用记录 | "不用记录" | ❌ |

### 9.2 优先级判断

| 优先级 | 场景 |
|--------|------|
| `high` | 用户明确要求、有截止日期、阻塞后续工作 |
| `medium` | 合理安排、重要但不紧急 |
| `low` | 可选功能、有时间再做 |

### 9.3 序号管理

- 序号在 TODO 根目录下统一递增
- 完成或取消的任务序号**不再重复使用**
- 删除任务时序号保留，不填补空缺

## 10. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 目录名 | 中文方括号 | `[归档]` |
| 文件前缀 | 中文方括号+序号 | `[待办]`、`[进行中]` |
| 序号 | 3位数字 | `001`、`002` |
| 分隔符 | `-` | `-` |
| 标题 | 中文或英文 | `科目一学习` |

**禁止**：
- ❌ 使用中文数字（如 `一`、`二`）
- ❌ 使用空格
- ❌ 使用特殊字符（除 `-` 外）

## 12. Checklist

创建或更新 TODO 时，检查：

- [ ] 文件名符合 `{状态}XXX-{标题}.md` 格式
- [ ] Frontmatter 包含所有必需字段
- [ ] `status` 与文件名前缀一致
- [ ] `updated_at` 已更新
- [ ] 序号在根目录下唯一
- [ ] 作业任务添加 `homework` 标签

## 13. 作业任务特殊规则

### 13.1 何时创建作业 TODO

| 场景 | 示例 | 是否创建 |
|------|------|----------|
| 用户提到作业 | "今天有作业要做" | ✅ |
| 用户要求记录作业 | "帮我记一下作业" | ✅ |
| AI 自己推断的作业 | 聊天中得知有考试 | ✅ |
| 一次性简单任务 | "改个变量名" | ❌ |

### 13.2 作业任务模板

```yaml
---
id: "003"
title: "高数课后习题"
status: "homework"
priority: "high"
created_by: "[life-agent]"
created_at: "2026-04-28T10:00:00Z"
updated_at: "2026-04-28T10:00:00Z"
due: "2026-04-30"
tags: ["homework", "高数"]
---

# [作业]003 - 高数课后习题

## 作业内容

- P45 习题1-3
- P46 习题5-7

## 要求

周四前完成
```
