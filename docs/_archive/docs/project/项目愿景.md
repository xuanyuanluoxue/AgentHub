# 项目愿景

> 统一 AI 工具的 Skill · Agent · 用户画像 · 记忆系统 四大共享生态

---

## 🔥 问题痛点

当前 AI 开发工具（OpenClaw、OpenCode、ClawHub、Claude Code 等）各自为战：

| 问题 | 说明 |
|------|------|
| **Skill 孤岛** | 各平台 Skill 无法互通，重复开发 |
| **Agent 割裂** | 每个工具需要单独配置 Agent |
| **画像丢失** | 切换工具意味着重新设置偏好 |
| **记忆不持久** | 每次对话都是新的开始 |
| **学习成本** | 每换一个工具就要重新学习其规范 |

---

## 💡 解决方案

AgentHub 提供**四大共享生态**，一次配置，到处使用：

### 1️⃣ 共享 Skill 库

- **统一格式**：YAML frontmatter + Markdown
- **跨平台兼容**：OpenClaw / OpenCode / ClawHub / Hermes 通用
- **依赖管理**：SemVer 版本控制
- **触发词机制**：智能匹配用户输入

### 2️⃣ 共享 Agent 库

- **类型定义**：Router（路由型）+ Specialist（专家型）
- **技能绑定**：自动加载所需 Skills
- **记忆策略**：短期/长期记忆可配置
- **跨工具共享**：同一 Agent 配置，多工具通用

### 3️⃣ 共享用户画像

- **工具无关**：纯文本 YAML，任何 AI 都能读取
- **永久携带**：换工具只需同步配置
- **自动同步**：通过符号链接保持多工具一致
- **隐私可控**：敏感信息本地存储

### 4️⃣ 共享记忆系统

- **多层记忆**：L0-L4 五级记忆架构
- **持久存储**：长期记忆跨会话保留
- **向量检索**：语义相似度匹配
- **智能上下文**：自动裁剪上下文窗口
