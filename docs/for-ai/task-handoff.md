# 任务交接规范

## 什么时候需要交接

当一个任务涉及多个 Agent 或需要跨会话传递时，使用任务交接文档。

## 交接文档模板

```markdown
# 任务交接

## 基本信息
- 任务ID: [唯一标识]
- 创建时间: [时间]
- 创建者: [Agent名称]
- 接收者: [Agent名称]

## 任务描述
[简要描述任务目标]

## 当前状态
- 状态: [进行中/已完成/已阻塞]
- 进度: [百分比或阶段]

## 上下文信息
### 用户信息
[相关用户画像]

### 已有结果
[已完成的部分]

### 待处理事项
[待完成的步骤]

## 关键路径
- [文件路径1]
- [文件路径2]

## 注意事项
- [特殊要求或约束]
- [已知问题]

## 后续步骤
1. [步骤1]
2. [步骤2]
```

## 交接流程

```
┌─────────────┐
│   Agent A   │ ← 创建交接文档
└──────┬──────┘
       │ 写文件
       ▼
┌─────────────┐
│ 交接文档    │ → 传递给 Agent B
└──────┬──────┘
       │ 读取
       ▼
┌─────────────┐
│   Agent B   │ ← 继续执行
└─────────────┘
```

## 文件命名

```
projects/
└── templates/
    └── task.md    # 交接文档模板
```
