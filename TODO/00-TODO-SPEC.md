# TODO 规范文档 v2.0

> 版本: v2.0 | 更新: 2026-05-15 | 适用: AgentHub

---

## 1. 概述

TODO 是 AgentHub 的任务追踪模块。使用文件夹区分状态，文件纯文字命名（无符号无括号）。

---

## 2. 目录结构

```
TODO/
├── README.md              # 使用说明
├── 00-TODO-SPEC.md        # 规范文档（本文件）
├── 待办/                  # 待开始的任务
├── 进行中/                # 进行中的任务
├── 作业/                  # 作业任务（独立分类）
└── 归档/                  # 已完成/已取消的任务
```

---

## 3. 状态定义

| 状态 | 含义 | 位置 |
|------|------|------|
| `待办` | 待开始 | `待办/` 目录 |
| `进行中` | 进行中 | `进行中/` 目录 |
| `作业` | 作业任务 | `作业/` 目录 |
| `归档` | 已完成/已取消 | `归档/` 目录 |

---

## 4. 文件命名规则

### 4.1 命名格式

```
{序号}-{标题}.md
```

### 4.2 规则说明

| 组成部分 | 规则 | 示例 |
|----------|------|------|
| 序号 | 3位数字，零填充 | `001` `002` `003` |
| 分隔符 | `-` | `-` |
| 标题 | 中文或英文，无特殊字符 | `科目一学习` `blog优化` |

### 4.3 示例

```
待办/001-科目一学习.md
进行中/002-博客优化.md
作业/003-高数课后习题.md
归档/004-旧任务完成.md
```

---

## 5. TODO 文件格式

### 5.1 YAML Frontmatter（必需）

```yaml
---
id: "001"
title: "科目一学习计划"
status: "pending"              # pending / in_progress / homework / completed / cancelled
priority: "high"              # high / medium / low
created_by: "dev-agent"
created_at: "2026-04-20T10:00:00Z"
updated_at: "2026-04-28T15:30:00Z"
---
```

### 5.2 必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 序号，3位数字 |
| `title` | string | 任务标题 |
| `status` | enum | 状态 |
| `priority` | enum | 优先级 |
| `created_by` | string | 创建者 ID |
| `created_at` | string | ISO 8601 时间 |
| `updated_at` | string | ISO 8601 时间 |

### 5.3 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `parent` | string | 父任务 ID |
| `children` | string[] | 子任务 ID 列表 |
| `tags` | string[] | 标签数组 |
| `due` | string | 截止日期 |
| `dependencies` | string[] | 依赖的任务 ID |

### 5.4 状态枚举

| status | 说明 |
|---------|------|
| `pending` | 待开始 |
| `in_progress` | 进行中 |
| `homework` | 作业任务 |
| `completed` | 已完成 |
| `cancelled` | 已取消 |

### 5.5 优先级枚举

| priority | 说明 |
|----------|------|
| `high` | 高优先级，用户明确要求/有截止日期 |
| `medium` | 中优先级，合理安排 |
| `low` | 低优先级，可选功能 |

## 7. 状态流转

```
待办/ ──开始──▶ 进行中/ ──完成──▶ 归档/
  │                    │
  └──取消──────────────┴──────▶ 归档/

作业/ ──开始──▶ 进行中/ ──完成──▶ 归档/
  │                    │
  └──取消──────────────┴──────▶ 归档/
```

### 7.1 操作规则

| 操作 | 文件操作 |
|------|----------|
| 创建待办 | 在 `待办/` 目录创建 `{序号}-{标题}.md` |
| 创建作业 | 在 `作业/` 目录创建 `{序号}-{标题}.md` |
| 开始任务 | 移动到 `进行中/`，更新 status |
| 完成任务 | 移动到 `归档/`，status 改为 `completed` |
| 取消任务 | 移动到 `归档/`，status 改为 `cancelled` |

## 8. AI 使用规范

### 8.1 何时创建 TODO

| 场景 | 示例 | 是否创建 |
|------|------|----------|
| 用户明确安排的任务 | "明天完成这个" | ✅ |
| AI 自己承诺的里程碑 | "我会在第三步完成后..." | ✅ |
| 需要多步骤完成的任务 | "这个需求需要5步" | ✅ |
| 一次性简单任务 | "改个变量名" | ❌ |
| 用户明确说不用记录 | "不用记录" | ❌ |

### 8.2 优先级判断

| 优先级 | 场景 |
|--------|------|
| `high` | 用户明确要求、有截止日期、阻塞后续工作 |
| `medium` | 合理安排、重要但不紧急 |
| `low` | 可选功能、有时间再做 |
